## 一、标签功能重构（核心）

### 现状问题
"计时"页只有一个标签输入框，每次都要手动输入，使用率极低（用户反馈"很鸡肋"）。

### 目标
像番茄ToDo那样：**标签是预设的、可视化的、点击即开始的**。

### 1. 标签管理（设置页新增"标签管理"区块）
- 每个标签字段：
  - `name`：标签名（必填，1-12 字符，全局不可重名）
  - `color`：颜色（8 预设色：红 / 橙 / 黄 / 绿 / 青 / 蓝 / 紫 / 粉）
  - `icon`：可选 emoji（不填也行）
- 操作：新增、编辑改名、改颜色、删除
- **删除时保留历史 tag 字符串**：删除标签只删 `tags` 表里那一行，**不动 `pomodoro_records` 表里 tag 字段**（历史可查，未来同名标签新建时历史记录会重新归属）

### 2. 标签数据存储
- 新建 `tags` 表（id, name, color, icon, created_at）
- 首次升级迁移：`init_db` 里加一段迁移 SQL，把 `pomodoro_records` 里所有非空的不同 tag 名自动插入 `tags` 表（color 默认 `#27AE60` 绿，icon 为空，created_at 用当前时间）

### 3. 计时页 - 标签选择器（替换原输入框）
- 原"标签 / 项目"输入框**完全删除**，替换为标签选择器
- 布局：横排彩色色块 + 标签名，自适应换行
- 默认选中：上一次用的标签（持久化在 settings 的 `last_tag` 字段，首次或无则空）
- 交互：
  - **单击标签** → 选中（高亮该标签的边框 + 轻微放大）
  - **双击标签** → 选中 + 立即开始专注（跳过点"开始"那一步）
  - 选中状态下点"开始"按钮 → 开始专注
  - 不选标签也可点"开始" → 按"无标签"记录（tag 字段为空）
- **专注进行中：标签锁定（变灰），不允许中途切换标签**（防止误操作）
- 计时圆环上方显示当前标签名 + 色点（如：`🟢 编程`）

### 4. 数据写入
- 表结构不变（已有 `tag` 字段），开始专注时把选中标签的 `name` 写入 `tag`
- 没选标签就 `tag` 字段留空字符串

## 二、统计卡片升级（看板 → 统计卡片页签）

### 现状
"项目标签明细"只是标签名 + 番茄数 + 时长 + 进度条的列表。

### 升级后

1. **顶部时间筛选器**：全部时间 / 本周 / 本月 / 今日
2. **摘要条**：「总计 N 个番茄 · Xh Ym」
3. **标签明细列表**（每行）：
   - 左侧：颜色圆点 + 标签名
   - 中部：横向进度条（按该标签时长占同时段总时长的比例）
   - 右侧：N 个番茄 · Xh Ym
4. **排序**：按时长降序
5. **空状态**：「还没有带标签的专注记录，去选一个标签开始吧 🎯」
6. **联动**：点击某个标签行 → 自动切到「热力图」页签 + 热力图筛选器自动选中这个标签
7. **附属小图**：每个标签行下方一个迷你"按日分布"柱状图（最近 7 天）
   - 柱子高度 = 当天该标签的专注时长
   - 鼠标悬停柱子显示：`日期 / Xh Ym`
   - 柱宽自适应，最多 7 根

## 三、热力图悬停升级

### 现状
悬停 tooltip 只显示：日期 / 专注时长 / 番茄数

### 升级后 tooltip 格式

```
2026-06-12
─────────────
总专注：2h 30m
番茄数：5 个
─────────────
● 编程   1h 30m（3 个番茄）
● 阅读     45m（1 个番茄）
● 数学     15m（1 个番茄）
```

- 标签按当日时长降序
- 圆点用标签的预设色（CSS 渲染，不是 emoji）
- 无记录的格子不弹 tooltip
- **重要**：如果某条历史记录的 tag 字符串在 `tags` 表里已不存在（即"幽灵标签"），tooltip 也要显示，颜色用默认灰
- 后端新增 API：`GET /api/stats/day-detail?date=YYYY-MM-DD` → 返回该日按 tag 聚合的明细（含已删除标签的 fallback 颜色）

## 四、数据持久化 + Access 接入

### 持久化（保持）
- SQLite，文件位置 = `sys.executable.parent`（exe 同目录的 `pomodoro.db`）
- v3 升级后旧数据完整保留

### Access 接入（方案 A：ODBC 链接）
- 应用代码**完全不动数据层**
- 用户在自己电脑装 [SQLite ODBC Driver](http://www.ch-werner.de/sqliteodbc/)（免费）
- Access 里"外部数据 → ODBC 数据库 → 链接表"指向 `pomodoro.db`
- 用户即可在 Access 里建查询、透视表、报表

### 应用里配套的"数据管理"区块（设置页新增）
- 显示当前数据库路径（只读文本框）
- 按钮：**复制路径** / **打开数据文件夹** / **导出 CSV** / **导出 JSON** / **完整备份**（下载 `pomodoro.db` 的 zip）
- 帮助文字（一段简短说明）：
  
  > 想用 Access 查看数据？请安装 SQLite ODBC Driver 后，在 Access 里通过 ODBC 链接到本应用的数据文件即可。

## 六、需要新增/修改的文件清单

### 后端 `app.py`
- `init_db`：新增 `tags` 表 + 迁移历史 tag 名到 tags 表
- 新增 API：
  - `GET /api/tags` → 返回所有标签
  - `POST /api/tags` → 新增标签
  - `PUT /api/tags/<id>` → 编辑
  - `DELETE /api/tags/<id>` → 只删 tags 表的记录，**不动 pomodoro_records**
  - `GET /api/stats/day-detail?date=YYYY-MM-DD` → 返回当日按 tag 聚合（含已删除标签的 fallback 灰色）
  - `GET /api/backup` → 下载 pomodoro.db 的 zip 备份
  - `POST /api/open-data-folder` → 触发资源管理器打开数据目录

### 前端 `static/`
- `index.html`：计时页加标签选择器容器；看板"统计卡片"区加联动入口；热力图 tooltip 升级；设置页加"标签管理"和"数据管理"区块
- `style.css`：标签色块样式、tooltip 样式、迷你柱状图样式
- `app.js`：
  - `STATE.lastTag` 持久化
  - 标签选择器渲染 + 单击/双击事件
  - 计时页专注中锁定标签
  - 热力图 tooltip 异步请求 `/api/stats/day-detail`
  - 看板统计卡片点击联动 → 切到热力图 + 应用筛选
  - 设置页标签 CRUD 操作
