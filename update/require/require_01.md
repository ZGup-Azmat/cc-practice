你需要帮我开发一个本地番茄钟应用，定位为"时间记录 + 数据看板"工具。请严格遵循以下需求，用清晰的模块化代码实现。

═══════════════════════════════════
一、核心计时功能
═══════════════════════════════════
1. 标准番茄钟循环：默认专注25分钟，短休息5分钟，每完成4个番茄钟进入长休息15分钟。所有时长可在设置中自定义。
2. 提供"开始"、"暂停"、"停止（放弃）"按钮，以及"跳过休息"选项。
3. 中途停止的番茄钟应保留实际专注时长记录，并标注为"未完成"。
4. 专注/休息结束时播放提示音，并弹出系统通知。
5. 开始专注前，允许用户选择或输入一个"标签/项目"（如"编程"、"阅读"、"数学"），也可留空。
6. 可选：专注结束后弹出一个微型反思窗口，让用户快速记录"专注度评分（1-5星）"和一行简短反思（可跳过）。

═══════════════════════════════════
二、每日学习总时长
═══════════════════════════════════
1. 实时计算并展示当日已完成的专注总时长（格式：X小时X分钟）。
2. 支持在设置中设定"每日学习目标时长"（可随时修改）。
3. 在界面显著位置用进度条/圆环展示当日目标完成百分比。
4. 达成目标时给出醒目提示（如弹窗 + 撒花动画）。

═══════════════════════════════════
三、数据看板（核心亮点）
═══════════════════════════════════
看板分为三个子模块，可在同一界面切换：

【模块A：热力图（类 GitHub 贡献图）】
- 每个单元格代表一天，用颜色深浅表示当日专注总时长。
- 支持「月视图」和「年视图」，一键切换。
  · 月视图：显示单月日历方格布局，可翻看不同月份/年份。
  · 年视图：类似 GitHub 的全年贡献矩阵，12个月横向排列。
- 鼠标悬停任意方格时，显示具体日期、当日总时长、完成番茄钟数。
- 颜色梯度：默认浅绿到深绿（可切换主题），空白（无记录）显示浅灰。
- 图例说明清晰可见。
- 当天新增记录后热力图自动刷新。
- 支持按"标签/项目"筛选热力图显示。

【模块B：趋势折线图】
- 展示「日/周/月」三个粒度的专注时长趋势。
- X轴为时间，Y轴为专注时长。
- 支持用虚线标注"每日目标线"，直观对比实际与目标。
- 可切换查看不同标签/项目的趋势。

【模块C：统计摘要卡片】
- 显示关键指标卡片：
  · 今日专注时长 / 目标完成率
  · 本周总时长 / 日均时长
  · 本月总时长 / 日均时长
  · 连续专注天数（streak）
  · 总累计专注小时数（全历史）
  · 最常使用的标签TOP3

═══════════════════════════════════
四、数据管理
═══════════════════════════════════
1. 所有数据存储在本地SQLite数据库（便于聚合查询）。
2. 提供数据导出功能，支持导出为CSV和JSON格式。
3. 数据表设计建议：
   - pomodoro_records：id, date, start_time, duration_minutes, 
     status(completed/abandoned), tag, focus_score, reflection
4. 所有数据纯本地，无任何网络请求。

═══════════════════════════════════
五、技术实现要求
- 待定（由用户后续指定技术栈）。
- 界面风格：简洁现代，使用卡片式布局，深色/浅色模式均需支持。
- 支持最小化到系统托盘，托盘图标显示当前计时状态。
- 所有动画和过渡流畅自然。
