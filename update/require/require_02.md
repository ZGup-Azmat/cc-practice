基于现有番茄钟项目源码，执行架构重构：彻底移除网页端，改造为纯Python Tkinter本地桌面应用，详细要求如下：

一、基础技术规则
1. 技术栈：仅使用 Python + 内置 Tkinter + SQLite，删除 Flask、pywebview、HTML、CSS、JS、浏览器相关所有代码、端口服务逻辑；
2. 保留原有数据库 pomodoro.db 及三张数据表（settings、pomodoro_records、tags），表结构完全不变，历史数据兼容；
3. 复用项目内 pomodoro.py 原有Tkinter原型代码，在此基础上迭代，不重新从零搭建；
4. 依赖仅保留：pystray、Pillow、PyInstaller，移除flask、pywebview等网页相关依赖；
5. 最终打包为独立本地exe，无网络、无本地服务、无浏览器依赖。

二、界面&功能复刻（完全保留原v4全部功能）
需要制作5个标签页窗口，和原网页端功能一一对应：
1. 计时页：番茄循环计时（25min专注/5min短休/15min长休可配置）、圆环进度展示、开始/暂停/停止/跳过按钮、番茄完成通知、反思弹窗、每日目标进度、关闭/最小化到托盘功能；
2. 任务页：CSV导入每日待办、日期切换、优先级分组（高/中/低）、任务预估番茄数、任务启动按钮、长期任务拆分、快速添加任务面板；
3. 看板页：热力图、趋势统计图、时长统计卡片、标签数据明细、时间段筛选功能；
4. 目标页：九大目标卡片、进度条、标签自动匹配累加进度、整体进度汇总；
5. 设置页：计时时长配置、每日目标设置、深浅色主题切换、数据库路径查看、数据导出、数据库备份。

三、通用交互要求
1. 界面整洁轻量化，支持浅色/深色双主题切换；
2. 所有SQLite读写操作增加事务、异常捕获、日志记录，保证本地数据安全，防止数据损坏；
3. 系统托盘功能保留：后台运行、右键菜单唤起主窗口、退出程序。

四、输出要求
1. 梳理最终目录结构，删除所有网页相关文件；
2. 给出完整可运行的Python代码，标注核心功能模块；
3. 提供PyInstaller打包命令，直接生成独立exe本地应用；
4. 附带基础运行、测试步骤。
